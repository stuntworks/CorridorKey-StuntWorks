## WHAT WAS FIXED THIS SESSION

### Bug 1: Zero PNGs written on BRAW clips
**Root cause:** The BRAW render queue uses a `time.sleep(1.0)` polling loop that blocks Fusion's event loop for 30-60 seconds. This permanently kills Fusion's PollTimer. The entire `_save_queue` + background thread architecture relied on PollTimer to drain saves — so nothing ever wrote to disk.

**Fix:** Added a synchronous BRAW processing path in `on_process_range`. When `_braw_frames_decoded` is populated (all frames pre-decoded in memory), skip the thread entirely. Process frames directly on the main thread using `save_output()` and `_do_import()`. Same pattern as single-frame preview. Returns before thread launch.

Location in code: around line 1728 in the installed file, inside `on_process_range`, right after the chroma warmup block.

### Bug 2: Off-by-one — one frame short
**Root cause:** Resolve's timeline end is exclusive. `src_end = ss + (out_f - cs)` missed the last frame.

**Fix:** `src_end = ss + (out_f - cs) + 1`

Location: around line 1636.
