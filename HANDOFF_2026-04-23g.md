## WHAT WAS FIXED THIS SESSION

### Fix 1 — SESSION_DIR was PID-based (SAM2 files lost after restart)
`corridorkey_session_{PID}` → `corridorkey_session` (fixed name, line 105).
Panel and viewer now always agree on the same temp dir across Resolve restarts.

### Fix 2 — braw-decode.exe crashed with 0xC0000005 (access violation)
Root cause: `CREATE_NO_WINDOW` removes the console handle. BlackmagicRawAPI.dll crashes on init without one.
Fix: replaced with `STARTUPINFO` + `SW_HIDE` — hides the window visually, console handle remains intact.
Location: `_try_braw_decode_exe()`, both the `-n` query and the decode Popen.

### Fix 3 — Viewer stayed open when PROCESS RANGE failed early
Root cause: `on_kill_viewer` was called AFTER the BRAW export block. If BRAW export returned None (early return), the kill never happened.
Fix: moved `on_kill_viewer(None)` to the very top of `on_process_range`, before any early returns.

### Fix 4 — Render queue audio kill removed
Removed the render queue fallback for BRAW entirely. If braw-decode.exe fails, panel shows a clear error and stops. No more `StartRendering()` = no more Focusrite audio resets.

### Fix 5 — Resolve hung on close (needed End Task)
Root cause 1: CUDA finalizer on Python exit blocks for 30-60s.
Root cause 2: `_viewer_proc.kill()` without `.wait()` left zombie holding GPU.
Root cause 3: PollTimer could fire during teardown against dead UI.
Fix: `os._exit(0)` after `disp.RunLoop()` returns; wait for viewer death in `on_close`; stop PollTimer first.

### Fix 6 — Diagnostic probes removed from _run() thread
37 `_probe(...)` lines removed. Thread is clean now.
