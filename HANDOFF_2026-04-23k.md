## AUDIO FIX — IN PROGRESS

### Root cause (confirmed):
Old BRAW testing code called `StartRendering()` many times + `CREATE_NEW_CONSOLE` on
braw-decode.exe spawned conhost.exe each time. Each one reset the Windows audio engine.
After enough resets, Resolve locked Focusrite in as the Fairlight ASIO device and it stuck.
Now Fairlight re-grabs the Focusrite on every Resolve startup → Focusrite Notifier pops up
→ driver resets → audio degrades.

### Fixes applied to code (already pushed):
- `StartRendering()` blocked for BRAW files (Path B guard)
- `CREATE_NEW_CONSOLE` → `CREATE_NO_WINDOW` on braw-decode.exe subprocess
- Both prevent the audio reset from happening again going forward

### Fix applied to system (this session):
- Focusrite ASIO buffer size changed from **128 → 256** in Resolve Preferences → System → Video and Audio I/O
- Full computer restart done to clear corrupted ASIO state

### What to test after restart:
1. Does the Focusrite Notifier still pop up when Resolve starts?
2. Does audio stay stable for 30+ minutes with Resolve open?
3. Does audio survive opening/closing the CorridorKey plugin?
