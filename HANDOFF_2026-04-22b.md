### INCOMPLETE C — tqdm monkeypatch not applied
Research confirmed: the stdout redirect around init_state is fragile.
Better fix: monkeypatch tqdm globally BEFORE SAM2 is imported anywhere.
Add this near the top of the module (after imports, before any SAM2 code):
```python
try:
    from functools import partialmethod
    from tqdm import tqdm as _tqdm_cls
    _tqdm_cls.__init__ = partialmethod(_tqdm_cls.__init__, disable=True)
except Exception:
    pass
```
This makes ALL tqdm progress bars silent — no IO, no stdout/stderr writes at all.
The stdout=stderr redirect around init_state can then be removed.

## KEY RESEARCH FINDINGS (from web scrape this session)

- OpenCV LZW TIFF: unfixable in pip wheels. Issue opencv#5662 closed "invalid".
  Never use cv2.imread for 16-bit LZW TIFFs — it silently returns None.
  
- tifffile is correct tool. Already installed (2026.3.3). Returns proper uint16.
  PIL has silent truncation bugs on 16-bit RGB TIFFs (Pillow#5248).
  
- tqdm disable=True is the ONLY safe suppression in Resolve. Skips ALL IO paths.
  `partialmethod` monkeypatch covers third-party libraries you don't control.
  
- Resolve fusionscript uses deprecated `imp` module — import lock not reentrant
  from background threads. Fixed in Resolve 18.6.5+. Workaround: import
  everything on main thread before spawning threads (which we already do).
  
- PNG NOT available via SetCurrentRenderFormatAndCodec. Use "dpx" or "exr"
  if TIFF ever needs replacing. PNG only works via Gallery/Stills API.
  
- BRAWconverter (gitlab.com/fnordware/BRAWconverter) — C++ CLI tool, converts
  BRAW→EXR/DPX/TIFF using public BRAW SDK. Mac only binary but source on GitLab.
  Could be reference for braw-decode-win Windows fix.

- pybraw (github.com/anibali/pybraw) — Python bindings, Linux only.
  Windows BRAW SDK DLL lives inside Resolve install folder, blocked externally.
