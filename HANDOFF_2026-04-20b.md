## NEXT BUG TO FIX: SAM2 MASK NOT APPLIED TO RENDERED FRAMES

### What Berto describes
- In the preview viewer, SAM2 clicks work visually (green=keep, red=remove)
- When SINGLE FRAME or PROCESS RANGE renders, the SAM2 mask is completely ignored
- Output looks identical to a chroma key with no SAM2

### Root Cause (confirmed)
`_merge_live_params()` in CorridorKey_Pro.py reads despill/despeckle from live_params.json
but NEVER reads SAM2 click points. The viewer stores clicks in its own process memory.
They never reach the panel's `sam_points` global.

`sam_points` global in panel: `{"positive": [], "negative": [], "frame": None}`
`generate_alpha_hint()` checks `sam_points["positive"]` and `sam_points["negative"]`
If both empty → falls back to chroma key. Always empty when clicks come from viewer.

### The Viewer (preview_viewer_v2.py)
File: D:\New AI Projects\CorridorKey\resolve_plugin\preview_viewer_v2.py (1501 lines)
- Stores SAM2 points as `self._sam_display_pts` — normalized 0..1 image coords
- `_save_live_params_now()` writes to SESSION_DIR/live_params.json
- Currently writes: despill, despeckle, despeckleSize
- Does NOT write SAM2 click points to live_params.json

### The Fix (two parts)

**Part 1 — viewer writes SAM2 points to live_params.json**
In preview_viewer_v2.py, find `_save_live_params_now()` (~line 779).
The payload dict currently has despill/despeckle values.
Add SAM2 points to the payload — in FULL FRAME pixel coordinates (not normalized).
The viewer knows the frame size (self._params width/height or the loaded image size).
Convert normalized coords → pixel coords before writing.

```python
# In _save_live_params_now(), add to payload:
sam_pos = [[int(x * frame_w), int(y * frame_h)] for (x, y, pos) in self._sam_display_pts if pos]
sam_neg = [[int(x * frame_w), int(y * frame_h)] for (x, y, pos) in self._sam_display_pts if not pos]
payload["sam_positive"] = sam_pos
payload["sam_negative"] = sam_neg
payload["sam_frame"] = self._sam_anchor_frame  # whatever the viewer stores as the clicked frame number
```

Check what self._sam_display_pts actually stores — grep for where it's appended in the viewer.
Check how the viewer gets the frame dimensions for coordinate conversion.

**Part 2 — panel reads SAM2 points from live_params.json**
In CorridorKey_Pro.py, `_merge_live_params()` (~line 308).
After reading despill/despeckle, also read sam_positive/sam_negative:

```python
if "sam_positive" in lp or "sam_negative" in lp:
    sam_points["positive"] = [tuple(p) for p in lp.get("sam_positive", [])]
    sam_points["negative"] = [tuple(p) for p in lp.get("sam_negative", [])]
    if "sam_frame" in lp:
        sam_points["frame"] = lp["sam_frame"]
    # Also switch alpha_method to SAM2 if points exist
    if sam_points["positive"] or sam_points["negative"]:
        out["alpha_method"] = 1
```
