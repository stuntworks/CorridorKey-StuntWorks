## THE MAIN REMAINING BUG — SAM2 VIDEO PROPAGATION FOR BRAW

**Problem:** Static SAM2 gate (one mask, applied to all frames) doesn't track performer movement. Looks wrong on any frame where performer has moved from the anchor frame.

**Root cause:** `braw-decode.exe` fails with exit code `3221225477`. When it fails, `braw_frames_dir` is never set, so SAM2 video propagation is skipped. The BRAW fallback path DOES produce TIFFs (Resolve seek+still method), but those TIFFs aren't being passed to `run_sam2_video_propagation`.

**The fix needed:**
In `on_process_range()`, when the BRAW fallback produces TIFFs, capture that temp directory path and set `braw_frames_dir` to it so SAM2 video propagation can use those frames.

Look for where the BRAW fallback writes TIFFs (log shows: `CK_BrawFB_298_443_XXXXXXXX`). That directory path needs to be captured and passed into the SAM2 propagation block at lines ~1957-1979.

**Why braw-decode.exe fails:**
Exit code 3221225477 = 0xC0000005 = Access Violation. The braw-decode.exe can't access the BRAW SDK outside of the Resolve process context. This is a known limitation — the BRAW fallback (Resolve seek+still) is the correct long-term path.
