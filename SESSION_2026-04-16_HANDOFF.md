## CRITICAL BUG — Fix First Next Session

**Viewer inline styles override QSS theme.** The _DARK_STYLE QSS was updated to Honeycomb (cyan), but `_build_ui()`, `_highlight_mode_button()`, and `_toggle_split()` in preview_viewer.py all have hardcoded inline `setStyleSheet()` calls with the OLD green/gray palette. In Qt, inline styles beat global QSS. This is why the viewer theme changes weren't visible.

**Fix:** Remove all inline setStyleSheet() color overrides. Let the global _DARK_STYLE QSS handle it. Only override the active mode button color dynamically.

## Architecture
- Panel (index.html) has sliders and sends updates to viewer via live_params.json
- Viewer (preview_viewer.py) polls live_params.json every 50ms and re-renders
- Refiner re-key is handled by the PANEL calling rekeyWithCurrentSettings() which runs ae_processor.py synchronously (blocks panel JS but not viewer Qt)
- Viewer starts fast because _ViewerColorUtils has numpy-only functions (no torch)
- The NN (torch) only runs in ae_processor.py which is a separate process
