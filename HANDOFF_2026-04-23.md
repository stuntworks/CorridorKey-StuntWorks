## NEXT STEP — TWO OPTIONS

### Option A: Decode numpy on main thread (cleanest fix)
Instead of passing BytesIO to thread, decode ALL frames to numpy uint8 BGR arrays
on the main thread BEFORE launching thread. Thread gets a plain list of numpy arrays.
No PIL, no tifffile, no file I/O in thread at all.

In `on_process_range` (main thread), after pre-reading BytesIO:
```python
_braw_frames_decoded = []
if _braw_tif_buffers and _has_pil:
    for _i, _buf in enumerate(_braw_tif_buffers):
        try:
            _buf.seek(0)
            _pil = _PILImage.open(_buf)
            _pil_rgb = _pil.convert("RGB")
            _arr = np.array(_pil_rgb)
            if _arr.dtype == np.uint16:
                _arr = (_arr >> 8).astype(np.uint8)
            elif _arr.dtype != np.uint8:
                _arr = (_arr.astype(np.float64) / float(np.iinfo(_arr.dtype).max) * 255.0).clip(0,255).astype(np.uint8)
            _braw_frames_decoded.append(np.ascontiguousarray(cv2.cvtColor(_arr, cv2.COLOR_RGB2BGR)))
            log(f"  Decoded frame {_i}: shape={_braw_frames_decoded[-1].shape}")
        except Exception as _de:
            log(f"  Decode frame {_i} failed: {_de}")
            _braw_frames_decoded = []
            break
    if _braw_frames_decoded:
        log(f"  All {len(_braw_frames_decoded)} frames decoded on main thread — thread uses raw numpy")
```

In thread, replace entire PIL/tifffile/cv2 read block with:
```python
if braw_frames_dir:
    fidx = tf - in_f
    if fidx < len(_braw_frames_decoded):
        frame = _braw_frames_decoded[fidx]
        _probe(f"frame {fidx}: shape={frame.shape} dtype={frame.dtype}")
    else:
        frame = None
```

This is the **definitive fix** — zero file I/O or library calls from the thread for TIFF reading.

### Option B: Diagnose PIL BytesIO hang
Add probe BEFORE and AFTER each PIL call:
```python
_probe(f"PIL open start frame {fidx}")
_buf.seek(0)
_pil = _PILImage.open(_buf)
_probe(f"PIL open done frame {fidx}: mode={_pil.mode}")
_pil_rgb = _pil.convert("RGB")
_probe(f"PIL convert done frame {fidx}")
```
If "PIL open done" appears but "PIL convert done" doesn't → libtiff decompression hanging.
If "PIL open start" appears but "PIL open done" doesn't → BytesIO parsing hanging.
