## ROOT CAUSE (CONFIRMED BY KIMI K2.5)
Windows Defender `Wdfilter.sys` hooks `IRP_MJ_CREATE` (every file open) synchronously.
Background threads in Resolve's `fuscript` host lack impersonation tokens → Defender
treats them as "low-trust injection targets" → scans every file open from thread.
Main thread is trusted. Background threads are not. No OS API to change this.

## THE FIX THAT WORKS (v0422-numpy)
Decode ALL TIFF frames to numpy uint8 BGR arrays on the MAIN THREAD before launching
the background thread. Thread receives plain numpy arrays — zero file I/O in thread.

Main thread flow (on_process_range):
1. Glob TIFFs → `_braw_tif_files_precomputed` (list of Path)
2. Read each TIFF into BytesIO → `_braw_tif_buffers` (list of BytesIO)
3. PIL decode each BytesIO → numpy uint8 BGR → `_braw_frames_decoded` (list of ndarray)
4. Open probe file fd → `_probe_file` (pre-opened handle)
5. Launch thread

Thread flow:
- Gets `_braw_frames_decoded[fidx]` directly — no PIL, no tifffile, no file open
- Feeds frame into AI (proc.process_frame)
- Saves PNG to output folder
